## 🦀 1. La app no compila (Tauri / Cargo)

### 🚨 Síntoma

```text
failed to read file '...permissions/path/autogenerated/._default.toml': stream did not contain valid UTF-8
```

> Causado por archivos AppleDouble (`._*`) en volúmenes externos no-APFS.

### ✅ Mitigación

```bash
bash scripts/mac/clean-appledouble.sh
# y reintenta
npm run tauri:build:app
```

> [!TIP]
> El repo ya redirige `target-dir` a `/tmp/chofyai-target` vía `.cargo/config.toml`. Si sigue fallando, revisa que no quede ningún `._*` en `apps/`, `src-tauri/capabilities/`, `src-tauri/icons/`.

### 🔍 Otros chequeos

- `cargo --version` (1.76+)
- `xcode-select -p` (debe imprimir un path)
- `npm --version` (10+)
- `bash scripts/mac/preflight-build.sh`

### 🔍 Qué revisar

| Punto | Cómo |
|:---|:---|
| 💾 `studio_home` correcto | Mira la barra inferior: `Disco · {path}` |
| 📂 Carpeta real existe | Botón 📁 en la tarjeta del tool |
| 📋 Log de arranque | Botón 📋 → busca tracebacks de Python o errores de puerto |
| ✅ `installed_if` realmente existen | `bash scripts/mac/doctor.sh "$STUDIO_HOME"` |
| 🔌 Puerto ocupado | `lsof -i :7860` (o el puerto de la tool) |

## 🎤 3. Qwen3-TTS falla al instalar

### 🔍 Puntos típicos

- ❌ No existe `python3.10` → `brew install python@3.10`
- 💾 Falta espacio en disco — los modelos pesan ~4 GB
- 🌐 Falla descarga de modelos — verifica conexión a HuggingFace
- ⚠️ El volumen de destino no es estable (USB intermitente)

## 💾 8. Studio Home en disco externo se desconecta

### ✨ Comportamiento esperado

ChofyAI usa `resolve_effective_home`:

- Si `/Volumes/ORICO/...` no está montado → cae a `~/ChofyAIStudio` automáticamente.
- La barra inferior muestra `⚠ Usando fallback`.

### 🔧 Si quieres forzar el fallback explícito

Edita `storage/state/settings.json`:

```jsonc
{
  "studio_home": "/Volumes/ORICO/ChofyIA/ChofyAIStudio",
  "fallback_home": "/Users/tuusuario/ChofyAIStudio-fallback"
}
```

## 📍 9. Reubiqué una tool pero la app sigue mirando la ruta vieja

### 🔍 Qué pasó

El override está en `settings.json → tool_overrides[<id>]`. Verifica que:

- La nueva ruta es **absoluta**.
- La nueva ruta existe y tiene los archivos del tool.
- No hay un proceso del tool corriendo (apaga primero, mueve después).

### ✅ Reset

Botón **↺ Reset ruta** en la tarjeta del tool, o manualmente:

```bash
# Edita settings.json y borra la entrada en tool_overrides
```

## 🧹 10. Quiero borrar una herramienta

```bash
bash scripts/mac/cleanup-tool.sh "/ruta/a/studio_home" "tool_id"
```

Ejemplo:

```bash
bash scripts/mac/cleanup-tool.sh "$HOME/ChofyAIStudio" "qwen3-tts"
```

> [!WARNING]
> Esto elimina toda la carpeta del tool incluyendo modelos descargados. Los logs en `logs/` se preservan.
